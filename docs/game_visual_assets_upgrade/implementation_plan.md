# Floria Link ビジュアルアセットおよび勝利条件の修正計画

ユーザーフィードバックに基づき、以下の3点の修正を行います：
1. **拠点（城）のグラフィックの修正となじませ**:
   - 3D風の城から、ゲーム背景に調和する **2Dカートゥーン/フラットベクター調の正面イラスト** に再生成します。
   - 周囲の影や不要な床部分を排除して完全に背景を透過させ、緑の芝生の上に浮かないように接地感を調整します。（※対応済）
2. **勝利条件の変更**:
   - 花畑の全開花による勝利を廃止し、**敵の城（腐海の巣）を破壊することのみを勝利条件** とします。花畑の開花は、引き続き資金や花蜜の獲得といったゲーム内の有利な要素として機能します。（※対応済）
3. **ゲーム終了後の画面揺れの廃止 (新規追加フィードバック)**:
   - ゲーム終了後（勝利または敗北してリザルト画面が表示されている際）に、画面が揺れたままになるバグ（状態更新が止まり減衰しないため）を修正します。
   - ゲーム終了時に `shake` 値を強制的に `0` にリセットし、かつ描画メソッド内でもゲーム中以外は画面を揺らさないように制限します。

## 提案される変更点

### 1. 城の画像の再生成と透過処理の改善 (対応済)
- `generate_image` を使用し、2Dフラットベクター調の城を再生成・透過加工して配置しました。

### 2. 勝利条件の修正 (`game.js`) (対応済)
- [game.js](file:///c:/Users/kojil/Documents/Dev/floria-link/public/js/game.js) の `tendPlot` メソッド内で、すべての花畑が開花したときに `this.end(true, ...)` を呼び出していた処理を削除しました。

### 3. ゲーム終了後の画面揺れの廃止 (`game.js`) (新規追加)
- [game.js](file:///c:/Users/kojil/Documents/Dev/floria-link/public/js/game.js) の `end` メソッドに `this.shake = 0;` を追加し、終了時に揺れを即座にリセットします。
- `draw` メソッド内の揺れ（`translate`）適用条件に、ゲーム中かつ未終了であること（`this.state === "play" && !this.ended`）を追加します。
