# Agent guide — Floria Link

このリポジトリでコードを編集するエージェント向けの短い指針です。

## プロダクト

- 静的配信のバニラ JS 2D ゲーム（育成 + 拠点防衛バトル）
- エントリ: `public/index.html` → `public/js/main.js`（または分割後の `app/`）
- サーバー API・認証・DB はない。永続化はブラウザ `localStorage` のみ

## 触る境界

| 領域 | 置き場の目安 | やってよいこと |
|------|----------------|----------------|
| 画面遷移・ブート | `app/` | ルーティング、ループ配線 |
| DOM UI | `ui/` | HUD・育成画面の描画とイベント |
| バトル計算 | `battle/` | シミュレーション（DOM に依存しない） |
| Canvas 描画 | `battle/render` | 描画のみ |
| 育成ロジック | `train/` | ターン進行・ステ計算・サポート |
| データ定義 | `data/` | 定数、カタログ読込、セーブリポジトリ |
| 共有 | `shared/` | audio / fx / clamp / events |

分割前のフラット構成（`game.js` / `training.js` 等）の場合も、上記の責務を混ぜないこと。

## やってはいけないこと

- 計画なしの大規模フレームワーク導入（Vite / TS は ADR を見てから）
- セーブキーやセーブ形式の破壊的変更（必要ならマイグレーションを書く）
- アセットの無関係な再生成・巨大バイナリの不用意な追加
- プランファイル（`.cursor/plans/`）の無断編集

## 検証

変更後は少なくとも次を手動確認する。

1. `npm start` で起動する
2. タイトル → 育成（1 ターン操作）→ 完了または中止
3. 出撃 → ユニット出撃 → 勝敗またはタイトル戻り

純関数の変更がある場合は `npm test` を実行する。
